## Theme E — API SDK 完整化

### 目標

讓外部開發者用熟悉的語言整合 staffKM，不必硬讀 OpenAPI 自己 hand-roll HTTP。

### 範圍（v4.5 first cut）

三個 SDK 都覆蓋以下 endpoint family：

- `auth` — login / refresh / me
- `workspaces` — list / create / get
- `knowledge` — KBs CRUD、document upload、hit_test
- `applications` — list / create / run
- `chat` — 含 SSE 串流（重點）
- `quota` — summary / set / list
- `billing` — users.list / detail / csv
- `plugins` — list / reload

### 非目標（明確排除）

- 不從 OpenAPI 自動 codegen（手寫覆蓋主 endpoint，自動化是 v4.x backlog）
- 不 publish 到 PyPI / npm / pkg.go.dev（local install only）
- 不做完整 type-safe（Go 不上 generic、TS 不每個 endpoint 都 typed）
- Async/await 在 Python 只先放骨架，正式版 v4.6 補

### 交付物（這個 PR）

- `packages/python/staffkm-sdk/staffkm/` — 新 module（保留舊的 `staffkm_sdk/`）
- `packages/ts/staffkm-sdk/` — 新 npm package
- `tools/sdk-go/` — 新 Go module
- 每個 SDK 至少 1 個 working example
- `docs/dev/sdks.md` — 統一說明
- `docs/plans/v4.5-roadmap.md`（本檔）

### 驗證

- Python `python -m py_compile` 全綠
- TypeScript `tsc --noEmit` 全綠（如環境裝得起）
- Go file structure OK（不要求 `go build`）
