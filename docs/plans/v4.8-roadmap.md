# v4.8 Roadmap — Usage-based metered billing (Theme H)

接 v4.7 G 的 Stripe 帳戶 + subscription，加上「按 token 計費」的
metered plan：nightly worker 把當期 usage 聚合，回報 Stripe Usage Records API。

## Scope

- **H1** `usage_reports` 表（unique key 防重送 + stripe_event_id）
- **H2** `core/billing.py::report_usage_to_stripe`（idempotent，DB 紀錄優先）
- **H3** `core/usage_report_worker.py`：每 6 小時聚合 `model_usage_logs`
  → 對 plan='usage' workspace 寫 usage_reports + call Stripe（placeholder）
- **H4** 接 agent main.py lifespan（inprocess）；arq mode 依現有 pattern 另佈

## Placeholder（明列）

- 真實 Stripe Usage Records 需要 `subscription_item_id`；目前只 log 並寫 DB，
  待 Stripe Workspace ↔ subscription item mapping 機制成型再接（v4.x 後續）。
- 沒有 hard stop：balance < 0 / quota exceed 只 log warning。

## 驗證
