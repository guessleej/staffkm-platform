# v4.7 Roadmap — Stripe billing (Theme G)

接 Stripe，把 v3.7+v3.8 已落地的 cost attribution / per-user billing
轉成可變現的訂閱 + pre-paid credits 流程。

## Scope

- **G1** Stripe SDK + config（lazy import，no-stripe env 不炸）
- **G2** alembic 0017：billing_accounts / billing_invoices / usage_reports / credit_ledger
- **G3** `/billing/me`、`/checkout`、`/portal`、`/webhooks/stripe`、`/invoices`、`/credits/ledger`
- **G4** Frontend Admin → Stripe Billing View（plan card + topup + invoices + ledger）
- **G5** Webhook routing（gateway PUBLIC_PREFIX `/api/v1/public/billing/`）

## Out of scope（明列）

- Coupons / promo codes
- Hard quota stop based on credit balance（balance < 0 只 log，不阻 request）
- Cancel-confirmation 自製流程（一律走 Stripe customer portal）
- Tax / invoice 法定格式（要看 Stripe Tax module，v4.x 後續評估）
- 真實 Stripe Usage Records API 的 subscription_item_id 映射 —— 留 placeholder（見 v4.8）
