# v4.9 Roadmap — AI-generated Workflow (Theme I)

承 v4.8 H 的 metered billing + v3.x 已穩固的 LLM provider 抽象，
讓使用者「以自然語言描述需求」就能產出可直接 import 進 LogicFlow 編輯器的
workflow JSON 草稿，大幅降低新建 workflow 的門檻。

## Scope

- **I1** `core/workflow_gen.py`：natural-language → workflow JSON pipeline
  - AVAILABLE_NODES catalog（~14 個常用 node types，給 LLM 看的 schema 摘要）
  - System prompt + few-shot 範例（請假審批）
  - LLM call（用 workspace 預設 provider/model；簡化先走 OpenAI 相容 endpoint）
  - JSON 解析（容錯 markdown code fence）
  - Schema 驗證（節點/邊一致性、有 start、type 在 catalog 內、key 唯一、edge 端點存在）
- **I2** `api/workflow_gen.py`：`POST /workflow-gen/generate`
  - require_member（workspace-scoped）
  - 回 `{ workflow, valid, errors, raw_response }`
- **I3** Gateway proxy：`make_proxy_router("workflow-gen")`
- **I4** Frontend：
  - `api/workflowGen.ts` client
  - `views/application/AIWorkflowGenModal.vue`（textarea + 結果區 + 下載 JSON / debug toggle）
  - 整合到 `WorkflowEditorView` toolbar：「✨ AI 生成」button → 開 modal
  - **簡化整合策略**：不直接灌進 LogicFlow runtime（避免動 lf.render 流程），
    產出後提供「下載 JSON」+「複製到剪貼簿」+「套用到畫布」（emit 給 parent；
    parent 走現有 `loadWorkflow` 的 `apiToLf` 後 `lf.render`）。
- **I5** 文件：`docs/dev/ai-workflow-gen.md`

## Out of scope

- 不做 streaming response（一次回完 JSON）
- 不做 multi-turn refine（「再加一個 email node」之類；v4.x 後續 backlog）
- 不做 RAG over docs（純 LLM + prompt 內 catalog）
- 不對 LogicFlow runtime 大改；UI 簡化路徑也接受純下載

## 驗證

- `python -m py_compile services/agent/app/core/workflow_gen.py`
- `python -m py_compile services/agent/app/api/workflow_gen.py`
- 三層 routing：agent main.py mount、gateway _generic_proxy 新增、gateway main.py mount
- prompt 內 few-shot JSON 範例可被 `json.loads` parse
- curl `POST /api/v1/workflow-gen/generate` 帶 X-Workspace-ID + JWT → 200

## Known limitations

- 複雜 workflow 仍需人工調整（特別是 condition 分支、loop body）
- 沒做嚴格 JSON Schema；目前 `_validate` 為 best-effort
- 完全靠 prompt-engineering 控品質，model 換掉 quality 會飄
