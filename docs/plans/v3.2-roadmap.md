# v3.2 Roadmap — Cost / Quota governance

> Status: planning（2026-05-18）
> Theme: **把 LLM 成本與 workspace 額度真正管起來**
> 估時：2 週、4 PRs

---

## 為什麼是這個主題

v3.x 主線是 production hardening。v3.0 給了 metrics、v3.1 收技術債，但 **LLM token 是真實出血**：
- 沒人能回答「上個月各 workspace 花了多少」
- workspace_quotas 表雖在、`check_quota()` helper 雖寫好，但**只有 chat_stream 一個地方接**
- pricing 完全沒落地 — `cost_usd` 欄位有、但 caller 必須自己算
- v3.0 的 `v3-llm-usage.json` dashboard 是 empty placeholder

## 現況盤點

| 元件 | 狀態 |
|---|---|
| `model_usage_logs` 表 | ✅ 有（cost_usd / prompt_tokens / completion_tokens / latency） |

### PR 1 — Pricing registry + cost calc helper（P1, ~3 天）

**目標**：把 per-model 定價變成 first-class data，cost 自動算。

**Schema**（走 alembic！v3.1 baseline 之後第一個真正 migration）
- `model_providers` 加：
  - `price_per_1k_input_usd: NUMERIC(10, 6)` nullable
  - `price_per_1k_output_usd: NUMERIC(10, 6)` nullable
- alembic revision `0002_provider_pricing.py`

**Seed data**
- 新檔 `services/agent/app/data/model_pricing.py`：dict 內建主流 model 定價（gpt-4o / claude-3.5-sonnet / gemini-1.5-pro / ollama-* = 0）
- bootstrap 時 `UPSERT` 進 `model_providers`（用 model name 對應）

**Helper**
- `services/agent/app/core/usage.py` 加：
  ```python
  async def calc_cost(session, *, provider_type, model, prompt_tokens, completion_tokens) -> float:
      """查 model_providers pricing 算 cost；查不到回 0."""
  ```

**Done when**
- `model_providers` 表所有列都有 pricing（或 NULL 表示「免費 / 自架」）
- helper 對 gpt-4o 10k in/2k out 算出 ~0.07 USD

### PR 2 — `record_usage` + `check_quota` 接到所有 LLM 呼叫（P1, ~4 天）

**目標**：每次 LLM call 都會 (a) 預檢 quota (b) 寫 usage log，無需 caller 操心。

**策略：在 provider base class 包**
- `services/agent/app/providers/base.py` 的 `chat()` / `chat_stream()` method 改成：
  ```python
  async def chat(self, ...):
      await check_quota(session, workspace_id)  # 超額 raise QuotaExceeded
      start = time.monotonic()
      try:
          result = await self._chat_impl(...)  # subclass 實作的搬到 _chat_impl
          await record_usage(session, UsageRecord(
              workspace_id=workspace_id,
              prompt_tokens=result.usage.prompt_tokens,
              completion_tokens=result.usage.completion_tokens,
              cost_usd=await calc_cost(session, ...),
              latency_ms=int((time.monotonic()-start)*1000),
              status="ok",
          ))
      except QuotaExceeded:
          raise HTTPException(429, "workspace 已達月用量上限")
      except Exception as e:
          await record_usage(session, UsageRecord(..., status="error", error=str(e)))
          raise
      return result
  ```

**接線清單**（grep `provider.chat(\|provider.acompletion(\|provider.chat_stream(`）
- chat_stream.py（已接 record_usage，改走新 base wrapper）
- workflow executor（每個 llm_node）
- application_agent（如有獨立呼叫）
- skill execution（如走 LLM）

**錯誤處理**
- `QuotaExceeded` 在 gateway proxy 層也要 pass through 429（HTTPException 自動處理）
- workflow run 內遇到 quota 超額 → workflow_run.status = "quota_exceeded" 而非 generic "failed"

**Done when**
- 手動跑 chat 一次，`model_usage_logs` 新增 1 列且 `cost_usd > 0`（用 paid model）
- 故意把 workspace quota 設 1 token → 第 2 次 chat 拿 429

### PR 3 — Admin Quota UI + Cost dashboard 接真資料（P2, ~3 天）

**前端**
- 新 view `apps/web/src/views/admin/QuotaView.vue`：
  - 列出所有 workspace + 當月用量 + 剩餘 + cap 設定 input
  - 即時 progress bar（token cap 走 70% 黃、90% 紅）
  - admin 可改 cap、save 寫回 `PUT /quota`（要 workspace scope，可能要新增 admin 跨 ws endpoint）
- 新 view `apps/web/src/views/usage/UsageView.vue`（user 看自己 workspace）：
  - 當月總覽卡片：tokens / cost / requests / quota 剩餘 %
  - 依日 / 依 model / 依 user 圖（小型 chartjs 或 recharts）

**後端**
- 新 endpoint `GET /api/v1/admin/quotas` — admin 列出所有 workspace quota + 用量
- 新 endpoint `PUT /api/v1/admin/quotas/{ws_id}` — admin 跨 workspace 設 cap
- `/usage/summary` 加 `by_model` / `by_day` aggregation（前端畫圖用）

**Grafana**
- `infra/monitoring/grafana/dashboards/v3-llm-usage.json` 從 placeholder 改實：
  - 連 PG datasource（grafana-postgresql-datasource 或 grafana provisioning）
  - panels：
    - 24h tokens / cost / requests timeseries
    - top-10 workspaces by cost (本月)
    - top-10 models by cost
    - quota utilization heatmap

**Done when**
- admin 登入看得到 `/admin/quotas`，可改 cap
- Grafana `v3-llm-usage` dashboard 顯示真實數字

## 不在 v3.2 範圍

- ❌ Pre-paid credits / billing（→ SaaS 主題、v4.x）
- ❌ Multi-currency（內部記帳 USD 就好）
- ❌ Per-user quota（先 workspace level，user level 留 v3.3+）
- ❌ Cost optimization 建議（「換成 gpt-4o-mini 可省 X」）→ 留 v4.x
- ❌ Slack / email alert（超額通知）→ 留 v4.x 跟 Triggers 結合
