# v4.2 Roadmap — 行銷站 + Landing / Pricing / Cases

> Theme B：純前端行銷站（marketing site）。
> 估時：1 週、原 4 個 PR 合併為 **1 個 PR**（純前端、無 backend、改動隔離在 `apps/marketing/`）。

## 範圍

升級 `apps/marketing/` 從單頁 HTML 改為 **Vue 3 SPA**（跟 `apps/web/` 同 stack：Vite + Vue Router + Tailwind），落地完整行銷站：

1. **Landing page** — Hero / Features grid（6 個能力）/ How it works / 客戶 logo / CTA banner
2. **Pricing** — 4 個 plan（Free Trial / Starter / Pro / Enterprise）+ feature 對比表 + FAQ
3. **Cases** — 3 個 case study（列表 + 詳細頁，`/cases/:slug`）
4. **About** — 團隊 / 使命 / 聯絡（`mailto:` 連結）

## 為何合併成 1 個 PR

- 純前端、無 DB migration、無 backend endpoint，change surface 完全 isolate
- 4 個頁面互相 reference（nav / footer / CTA 連結），分 PR 反而 review 累
- 不影響 `apps/web/` 主 app、不動 gateway / 任何 service

## 不做（留 backlog）

- ❌ i18n（純 zh-TW 寫死，en / zh-CN 留 v4.x）
- ❌ Headless CMS（Sanity / Strapi）— 內容寫進 `src/data/*.ts` source-of-truth
- ❌ SSR / SSG — 純 SPA 即可（marketing 站 SEO 需求時再升 Astro / Nuxt）
- ❌ Contact form 後端 — 用 `mailto:hi@staffkm.dev`
- ❌ 動 main app（`apps/web/`）

## 部署

- `cd apps/marketing && pnpm build` 產 `dist/`
- compose 加 `marketing` service（profile `marketing`，nginx 靜態服務，port 8081）
- prod 由 Caddy reverse proxy（root → marketing，`/app/*` → ui）— 留 v4.3 整合

## CTA 串接

- "免費試用 14 天" / "登入" → `<a href="/signup">` / `<a href="/login">`（假設 same domain，跟主 app）
- 如需 cross-domain，後續加 `VITE_APP_URL` env override

## 驗證

- `pnpm install && pnpm build` 通過
- 5 個 view 都能 navigate（Home / Pricing / Cases / CaseDetail / About）
- TypeScript 編譯 OK

## 下一步（v4.3 候選）

- i18n（zh-TW / en）
- Blog（MDX 或 Markdown）
- SEO（meta tags / og:image / sitemap）
- Analytics（Plausible / Umami）
