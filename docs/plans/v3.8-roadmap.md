## 4 PR 拆解

### PR 1 — Workflow conversation_id pass-through（P1, ~1 天）
v3.7-P1 把 conversation_id 加進 metering 全鏈，但 workflow executor 暫保持 None。
這版補上：
- `WorkflowExecutor.__init__` 加 `conversation_id` keyword-only
- `trigger_dispatcher` 從 `event_triggers.application_id` 對應拿（暫 None，trigger-fired workflow 沒 chat context）
- 從 chat embedded workflow（application + workflow 同一個）：caller (chat_stream) 傳進
- executor 內 7 個 meter call site 全部帶 `conversation_id=self.conversation_id`

### PR 2 — Per-user billing UI（P2, ~3 天）
admin 後台加「Billing」分頁：
- 列每個 user 當月 cost / tokens / 對話數
- 點 user → 展開：top conversations / top features / 每日趨勢
- 匯出 CSV
- API:
  - `GET /admin/billing/users` — 所有 user 當月細項
  - `GET /admin/billing/users/{user_id}` — 單一 user 詳情

### PR 3 — Multi-judge averaging（P2, ~2 天）
`rag_eval.py` 支援多個 judge 平均：
- `--judge-models gpt-4o-mini claude-3-haiku-20240307`（multiple）
- 每 query 對每個 judge 都評一次、取平均 + 個別分數
- output 加 `judge_consensus` 欄（多 judge 的標準差也記錄）
- 文件補多 judge 選擇建議

### PR 4 — Query plan analyzer（P2, ~3 天）
slow_query listener 偵測到 slow → 自動跑 EXPLAIN ANALYZE：
- 新 service: `core/query_explainer.py`
- slow query event 觸發 → 把 statement 用 `EXPLAIN (ANALYZE, FORMAT JSON)` 跑一次
- 結果寫進 `slow_query_explains` 表 + 加進 OTel span（preview）
- admin API `/admin/slow-queries` — list + 點 row 看完整 plan
- 前端 `SlowQueriesView.vue` — table + 展開 plan tree
