# v5.0 Roadmap — Active-Active Multi-Region (Theme K)

> 狀態：scaffolding only — 真實 active-active GA 排在 **v5.x** 系列。
> 撰寫日期：2026-05-18

## 1. 為何 v5.0 是 major release？

v4.0 落地了 **active-passive multi-region**（PG streaming replica + read routing
+ MinIO secondary）；v5.0 把目標推到「**多寫節點同時可寫**」的 active-active
拓撲。這牽動：

- DB schema（要存 region pin、衝突記錄）
- 寫請求路由（要能 forward / redirect）
- 衝突偵測 + 解決框架（CRDT / LWW）
- ops 流程（multi-region failover testing、conflict alarm）

任何一點都會破壞 v4.x 的單寫節點假設，故走 **major**。

但 — 真實 production active-active 需要的「跨 region PG logical replication」
是大量 ops 工作（managed Aurora Global / Citus / Postgres-BDR / Bucardo 其
一）；單純在程式碼層交付是不夠的。

故 v5.0 **只交 scaffolding**：DB schema + region routing middleware（預設
disabled）+ admin API + CRDT helper stub + 完整架構文件。真正打開 multi-region
寫流量由 v5.1+ 系列 ops PR 帶。

## 2. v5.0 Scope（這個 release）

| 模組 | 內容 | 預設狀態 |
|---|---|---|
| `alembic 0019` | `workspace.primary_region` 欄位、`regions` 表、`region_conflict_log` 表 | always-on（schema 增量、無 breaking） |
| `RegionRouterMiddleware` | 寫請求若打到非 primary region → 308 redirect | **disabled**（`MULTI_REGION_ENABLED=false`） |
| `/api/v1/admin/regions` | regions registry CRUD | admin-only |
| `/api/v1/admin/conflicts` | conflict log viewer + manual resolve | admin-only |
| `/api/v1/admin/workspaces/{id}/region` | 把 workspace 釘到 primary region | admin-only |
| `app/core/crdt.py` | LWW resolver + G-Counter stub | helper only（未接 hot path） |
| Frontend `RegionsView.vue` | regions + conflicts 簡單 admin UI | nav `globe` icon |
| Docs | `docs/deploy/active-active.md`、`docs/upgrade/v4-to-v5.md` | — |

## 3. 不做（明確留 v5.x）

- ❌ 不真做跨 region PG logical replication 設定
- ❌ 不做 region failover automation（DNS swap / health-based reroute）
- ❌ 不真實現 CRDT 完整 algebra（OR-Set / RGA / G-Counter map）
- ❌ 不在 compose 模擬多 region（compose 是單 host）
- ❌ 不破壞既有 v4.x 單 region deploy（middleware 預設 off）

## 4. v5.x 後續計畫

## 5. 升級路徑

見 [`docs/upgrade/v4-to-v5.md`](../upgrade/v4-to-v5.md)。簡言之：

1. 跑 alembic upgrade head（schema 增量，秒級）
2. 預設 `MULTI_REGION_ENABLED=false` → 行為等同 v4.x
3. 想試 region routing：起兩個 region instance、設 `REGION_ID` + `REGION_<X>_URL` env、admin API 註冊 regions、把 workspace 釘 primary
4. 真實 active-active write GA 等 v5.1+

## 6. 風險登記

| 風險 | 緩解 |
|---|---|
| 客戶誤以為 v5.0 = 已 GA active-active | 文件 + CHANGELOG 明標 scaffolding-only |
| 寫流量被誤導 308 redirect、出 redirect loop | middleware 預設 off；只有 admin 明確設定才會啟動 |
| CRDT LWW 在 wall clock skew 下 silently lose write | v5.0 文件聲明只給 reference impl，不接 hot path |
