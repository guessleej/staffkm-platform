# v4.4 Roadmap — Theme D：staffkm-cli 進階 + Terraform provider scaffold

> Status: D1–D2 合併進同一個 PR 落地（IaC 化平台管理）。

## 動機

到 v4.3 為止，平台所有 admin 操作只能透過：
1. Web UI（apps/web）
2. 直接 curl gateway endpoint（手動帶 JWT）
3. `staffkm-cli`（v3.x 起內建，但只覆蓋 ping/apps/chat/usage/quota 5 個面向）

DevOps / SRE 想 script 化（CI 部署、跨 workspace bulk 操作、IaC declarative state）目前**只能自己包 shell script + curl**。v4.4 Theme D 目標：把 `staffkm-cli` 擴成完整 IaC 工具，並建立 Terraform provider 的骨架，未來 v4.5 SDK 完成後接 CRUD。

## 範圍（這個 PR）

| 子項 | 範圍 |
|---|---|
| **D1: admin-cli 擴充** | 新增 8 個 command group：`login` / `workspace` / `app` / `kb` / `user` / `plugin` / `observability` / `billing`；保留現有 `ping/apps/chat/usage/quota`。credentials 走 `~/.staffkm/credentials.json`（取代純 env） |
| **D2: Terraform provider scaffold** | `tools/terraform-provider-staffkm/` — provider schema + 2 個 example resource（workspace / kb）CRUD placeholder + examples/main.tf |
| **D3: 文件** | `docs/dev/cli.md` ~80 行 |

## 不在範圍（v4.5+ backlog）

- **Terraform CRUD 真接 API**：本 PR 只 scaffold + TODO，等 Go SDK（v4.5）完成後接
- **PyPI / Homebrew 發布**：v4.5 backlog
- **CLI shell completion script**：click 自帶 `_STAFFKM_CLI_COMPLETE=zsh_source`
- **CLI 互動式 TUI**（rich.prompt 之外）

## 架構

```
┌───────────────────────────────────────────────────────────┐
│  DevOps / SRE                                             │
│  ├─ staffkm-cli login --base http://… --email … --pass    │
│  │     └─ POST /auth/login → token → ~/.staffkm/cred.json │
│  ├─ staffkm-cli workspace switch <id>                     │
│  ├─ staffkm-cli kb upload <kb-id> ./policy.pdf            │
│  └─ staffkm-cli billing report --month 2026-05 --csv      │
│                                                           │
│  Terraform                                                │
│  └─ resource "staffkm_workspace" "demo" { name = "…" }    │
│        └─ tools/terraform-provider-staffkm (scaffold)     │
└───────────────────────────────────────────────────────────┘
```
