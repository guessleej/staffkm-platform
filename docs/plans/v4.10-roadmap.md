# v4.10 Roadmap — Theme J: Workflow Marketplace

> 跨 org 公開 template gallery — 任何人不需登入都能瀏覽，登入後可 install。

## 背景脈絡

| 版本 | 範圍 |
|---|---|
| v2.5-C | Workspace template marketplace（同 platform 內跨 workspace 分享）— `workspace_app_templates.is_public` / `install_count` / `/api/v1/workspace/{ws}/app-templates/marketplace` |
| v4.1 A | 5 starter packs（platform-curated）|
| v4.3 C | Plugin marketplace（`tools/plugin-marketplace/index.json`）|
| **v4.10 J** | **跨 org public gallery** — 不需登入即可瀏覽 / 評分 / 看詳情 |

## 範圍

### In scope

- 擴充 `workspace_app_templates`：
  - `publisher_name / publisher_url / cover_image_url`
  - `category / tags(JSONB) / license`
  - `verified` 旗標（staffKM team 手動加）
  - `rating_avg / rating_count`
- 新增 `template_ratings` 表（user-by-user，PK=(template_id,user_id)）
- 新 public API `/api/v1/public/marketplace/*`：
  - `GET ""` list + filter + sort + pagination
  - `GET /categories`
  - `GET /{id}` detail + recent 5 ratings
  - `POST /{id}/rate` (需登入 → 在 router 內檢查 user_id)
- Frontend：
  - `MarketplaceHomeView.vue` — search / category / sort / card grid
  - `MarketplaceDetailView.vue` — detail + install CTA + rating form
  - 路由 `/marketplace` 與 `/marketplace/:id` (`meta.public = true`)
  - LoginView 加「探索 marketplace」link
- 文件：`docs/dev/workflow-marketplace.md`

### Out of scope（留 v4.x+/v5.x）

- 不做 download tarball — template 本身就在 DB `schema_json`
- 不做 versioning（v4.x backlog）
- 不做 fork / star（GitHub-like 留 v5.x）
- 不對 v2.5 endpoint 做 schema 改動，只 ADD COLUMN

## 三層 routing checklist（CLAUDE.md §7）

1. agent main.py: `include_router(marketplace.router, prefix="/api/v1/public/marketplace")`
2. legacy_bridge: 不需要（public path 不走 workspace 翻譯）
3. gateway main.py + `_make_public_marketplace_router()`

`PUBLIC_PREFIXES` 已含 `/api/v1/public/` — 不需動。
