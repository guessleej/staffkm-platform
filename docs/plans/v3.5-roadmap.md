3. ❌ Run step persistence — 每個 node 的 input/output 沒寫進 DB（debug 看不到）
4. ❌ Run history UI — 前端沒有 step-by-step viewer

## 4-PR 拆解

### PR 1 — Run step persistence + history API（P1, ~3 天）

**目標**：每個 node 執行的 input/output 寫進 DB，讓 v3.5 後續 UI / debug 有資料可看。

- alembic `0007_workflow_run_steps`：
  ```sql
  CREATE TABLE workflow_run_steps (
      id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      run_id          UUID NOT NULL,        -- FK 到 event_trigger_runs.id
      step_index      INT NOT NULL,         -- 執行序
      node_key        VARCHAR(64) NOT NULL,
      node_type       VARCHAR(32) NOT NULL,
      status          VARCHAR(16) NOT NULL, -- ok / error / skipped
      input_snapshot  JSONB,
      output_snapshot JSONB,
      error           TEXT,
      attempts        INT NOT NULL DEFAULT 1,
      latency_ms      INT,
      started_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
      finished_at     TIMESTAMPTZ
  );
  CREATE INDEX idx_wrs_run ON workflow_run_steps(run_id, step_index);
  ```
- executor 改：每個 node 跑完寫一筆 step；error 也寫；retry attempt 用 attempts 欄
- input/output snapshot 截斷 4KB 防爆量
- 新 API `GET /api/v1/workspace/{ws}/applications/{app_id}/runs/{run_id}/steps`

### PR 2 — `human_approval` node + resume API（P1, ~5 天）

**目標**：workflow 跑到 human_approval 暫停、生 approval link、等 admin 點同意 → resume。

- alembic `0008_workflow_approvals`：
  ```sql
  CREATE TABLE workflow_approvals (
      id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      run_id        UUID NOT NULL,
      node_key      VARCHAR(64) NOT NULL,
      status        VARCHAR(16) NOT NULL DEFAULT 'pending', -- pending/approved/rejected
      requester     VARCHAR(128),                            -- workflow trigger 來源
      approver_id   UUID,                                    -- 點同意的 user
      approver_note TEXT,
      payload       JSONB,                                   -- 給 admin 看的 context
      created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
      resolved_at   TIMESTAMPTZ
  );
  CREATE INDEX idx_wa_run ON workflow_approvals(run_id);
  CREATE INDEX idx_wa_pending ON workflow_approvals(status) WHERE status = 'pending';
  ```
- `event_trigger_runs` 加 `paused_at` / `resumed_at` 欄
- executor 加 `_exec_human_approval`：寫 approvals row（status=pending）+ raise `WorkflowPaused` exception
- trigger_dispatcher catch `WorkflowPaused` → status = `paused`、不更 finished_at
- 新 API：
  - `GET /api/v1/workspace/{ws}/applications/{app_id}/approvals?status=pending` — list
  - `POST /api/v1/workspace/{ws}/approvals/{id}/approve` — admin 同意 → 觸發 resume worker
  - `POST /api/v1/workspace/{ws}/approvals/{id}/reject`
- 新 `core/workflow/resume_worker.py` 背景 loop：每 30s 掃 status='paused' run 且對應 approvals 已 approved → 從 paused node 之後續跑
- 前端 view `views/admin/ApprovalsView.vue`：pending list + approve/reject button + 顯示 payload context
- LogicFlow node 樣式 + 預設 config：`{ approver_role: 'admin', payload_template: '{{user_input}}' }`

### PR 3 — `sub_workflow` node（P2, ~3 天）

**目標**：workflow step 呼叫另一個 workflow（同 workspace 同 application 或跨）。

- executor 加 `_exec_sub_workflow`：
  ```python
  sub_app_id = config["sub_application_id"]
  sub_input = render_template(config["input_template"], context)
  sub_executor = WorkflowExecutor(workspace_id=..., application_id=sub_app_id, ...)
  async for ev in sub_executor.execute(user_input=sub_input):
      # nested events 加 prefix 不污染外層
      yield {"event": f"sub.{ev['event']}", "data": ev["data"]}
  # 把 sub 的最後一個 result 塞回外層 context.{output_variable}
  ```
- 防無窮遞迴：executor 加 `_depth` 參數，>3 raise
- LogicFlow node 樣式 + 預設 config：`{ sub_application_id: '', input_template: '{{user_input}}', output_variable: 'sub_result' }`

### PR 4 — Run history UI（P2, ~3 天）

**目標**：admin 看 workflow run 詳細 step-by-step。

- 新 view `views/admin/RunHistoryView.vue` (或在 ApplicationDetail / TriggersView 內加 drawer)：
  - 列出最近 N 個 run（從 `event_trigger_runs`）+ status badge + tokens / cost
  - 點某 run → 展開 step timeline
    - 每 step 顯示 node icon + node_key + status + latency
    - 點 step → 顯示 input/output snapshot（pretty-print JSON）
    - error step 顯示 error message + attempt 數
- 加 nav `/admin/run-history` 或從 triggers view 跳入
- replay button（v3.5 placeholder，實作 phase2）
