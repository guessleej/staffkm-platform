# Jobs-to-be-Done & 痛點對照表 v1

> 把 staffKM 的功能對應到實際「工作要被完成的事」與「現況的痛」。
> 用來校正開發優先序：每個功能必須回應至少一個 JTBD 與一個痛點。

| JTBD ID | When（情境）                                | I want to（行為）                       | So I can（目的）                       |

## 4. 取捨原則

當功能必須砍：
1. **沒對應到任何 JTBD** → 立刻砍
2. **JTBD 只有單一 persona × 單一 ⭐** → 延後到下版本
3. **JTBD ⭐⭐ 以上但要破壞既有體驗** → 走 RFC 討論
4. **JTBD ⭐⭐⭐** → 是必交付項目，不討論
